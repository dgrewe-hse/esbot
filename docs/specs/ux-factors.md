# UX (Experience) Factors
## 1 Clartity/Comprehensibility
### Definition
Clarity and comprehensibility mean that the user can operate the chatbot using only their existing skills, without needing to learn new conventions specifically for the tool. All buttons and functions should behave as expected, their labels should be precise and intuitive, and the interface should provide clear visual feedback when actions are performed.

### Relevance
This factor is particularly relevant for ESBot users, who are computer science students already familiar with many other chat-based tools. If ESBot is less intuitive or harder to understand than tools they already use, they are likely to abandon it quickly, as users generally do not want to invest time in learning how to use a tool before they can benefit from it.

### Example
A concrete example where clarity and comprehensibility are critical is the question input process and navigation. Users should immediately recognize where to type their questions, how to access that input field, and how to switch between previous chat histories without confusion or additional learning effort.

## 2 Feedback quality
### Definition
Quality and comprehensibility of the AI-generated responses to user inquiries, particularly with regard to technical correctness and traceability.

### Relevance
The user aims to learn as effectively and sustainably as possible. It is therefore crucial that the AI's answers are not only correct, but also convey content in a structured, understandable, and learning-conducive manner. High-quality answers actively support the learning process, reduce frustration, and promote a deeper understanding.

### Example
A user submits their own solution to a task. The AI does not only evaluate it as right or wrong, but provides targeted feedback: it explains the underlying principles, points out potential sources of error, and adds further exercises or content with which the user can deepen their understanding.

## 3 Usability
### Definition
The ability of a system to provide the specific functions and features required by the user to achieve their goals.

### Relevance
For a learning assistant, utility is the "make or break" factor; if the AI doesn't provide pedagogical value, the system is useless regardless of how pretty the UI is.

### Example
The student asks for a practice task. ESBot generates a specific software testing exercise (Utility) and provides corrective feedback on the student's answer (Utility), rather than just giving a generic definition of the term.

## 4 Efficency
### Definition
The speed and accuracy with which a user can complete a specific task using the system.

### Relevance
Students often use assistants for quick clarifications; long wait times for AI inference or complex navigation would hinder the learning flow.
Everything else results in happy users.

### Example
Instead of searching through lecture slides, the student types "/quiz" or a similar prompt. ESBot immediately pulls the stored learning context and generates a relevant question. The student receives instant feedback, completing a study cycle in seconds rather than minutes.

## 5 Trustability / Transparency
### Definition
Trust and transparency of AI responses means that users can rely on the AI's answers because the generation process is clear and sources are given.

### Relevance
Trust and transparency of AI responses are important for ESBot users because they rely on accurate and understandable information for learning and coding. Clear explanations and visible sources help them follow the logic, detect mistakes, and improve their problem-solving skills. At the same time, transparency allows them to critically evaluate the answers and use them responsibly in their academic work.

### Example
A CS student asks ESBot to explain a machine learning concept for a report. ESBot provides a concise explanation and includes references to textbooks or research papers. Because the sources are given, the student can verify the information and properly cite it in their assignment, ensuring academic integrity.
