### Project Setup
#### System Environment: Linux Mint (VM)
The project is developed on Linux Mint as operating system, which is hosted as virtual machine with VMWare Workstation Pro. 
This ensures that every developer works under the same conditions and makes troubleshooting of system based errors easier.
Working on an virtual machine additionaly ensures security and keeps the host system clean. 

#### IDE: Intellij, VSCode, VIM, Nano
The IDEs are chosen based on previous experiences and personal preferences. 

#### Version Control: Git (GitHub)
Git as versioning tool is mandatory for the Lab.
The initial projekt is distributed via GitHub and has to be forked for the lab.

### Technologie Stack
#### Frontend: Angular v21.0, Node.js v24.0.0, TypeScript
Angular was chosen to gain first practical experience with the framework and compare it with frameworks used in previous lectures.
Additionally it is a widely used and well documented open source framework Angular, which offers a lot of free information, making it easier to find solutions and tutorials.

As part of the Node.js environment npm and Node.js are needed for developing an Angular frontend. Also Angular is written for typescript, unlike React, were you can chose between typescript and javascript.

Lastly stable and supported versions were selected.

#### Backend: Spring Boot v4.0.4, Java v21, Maven
For the backend Spring Boot was chosen because our team is familiar with the framework due to prior lections and projects. Additionally it is well documented and easy to set up with spring initializer.

#### Database: PostgreSQL v17.9
Due to other lectures and projects, our team has previous experience and knowledge with this technology. 
It is also widely used and open source, therefore easily accessible. 

#### Virtualisation: Docker v29.3
Previous experience in our project, so we decided to use it again
Furthermore it is widely used, with many tutorials available and a technology we are likely to need in the future.
Open source makes it free to use and easy accessible.

#### Security (optional) & AI-Model: to be implemented later
Implementing dedicated Security Frameworks and LLM-integration is only relevant later on in the project. 
Within the scope of this lab: it can also be easily simulated/mocked for testing purposes. Therefore full implementation is not a priority during the early stages of the project.
